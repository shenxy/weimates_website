## Context

首页当前内容已经形成“价值主张 → 平台定位 → 产品界面 → 核心优势 → 应用场景 → 安全治理 → 权威背书 → 预约演示”的完整叙事，但缺少真实客户采用信号。项目中已有 `assets/csg.svg` 与 `assets/jnu.svg` 两个客户 logo 素材，适合直接用于客户背书展示。

## Goals / Non-Goals

**Goals:**
- 在首页加入两个重要客户的品牌标识，增强潜在客户对 weiMates 落地能力的信心。
- 保持客户背书与“权威背书”区块的信息边界：客户采用代表市场与场景验证，荣誉背书代表专业认可与测试认可。
- 确保桌面端和移动端都能清晰呈现客户 logo 与简洁说明，不产生拥挤、错位或过度营销感。

**Non-Goals:**
- 不新增完整客户案例详情页。
- 不声明未经确认的量化收益、合同细节、上线范围或具体使用部门。
- 不改变首页封面标题、平台定位和核心优势的既有价值主张。

## Decisions

1. 新增独立的客户背书区块，而不是把客户 logo 放入“权威背书”。
   - 原因：客户采用和奖项/测试属于不同信任来源，混放会降低信息清晰度。
   - 备选方案：把两个 logo 放进现有 honors-grid；该方案实现更快，但会让“权威背书”语义变宽，后续扩展案例也不清晰。

2. 区块位置放在“权威背书”之后、“预约演示”之前。
   - 原因：访客先理解产品能力与治理可信度，再看到客户采用信号，最后进入转化动作，信息节奏更自然。
   - 备选方案：放在首页封面下方；该方案曝光更早，但会打断现有“平台定位”承接，不利于先建立产品语境。

3. 文案采用“携手重点客户，共建高价值场景”的稳健表达。
   - 原因：当前需求是增强信心，并表达重点企业、高校客户对平台的信任，以及与 weiMates 共同验证高价值场景的关系；在未提供可公开披露业务细节的情况下，应避免夸大到“标杆案例”“深度落地成果”等未经证实表述。
   - 备选方案：使用强案例标题与长描述；该方案营销感更强，但对素材和授权细节要求更高。

4. 使用静态 HTML/CSS 实现，不引入新依赖。
   - 原因：当前官网是静态页面，新增区块可直接复用现有 reveal 动效、section-shell、section-heading 等样式体系。
   - 备选方案：引入轮播或交互卡片；两个客户数量较少，轮播会增加复杂度且降低可见性。

## Risks / Trade-offs

- [Risk] 客户名称或 logo 公开授权范围未明确 → Mitigation：只使用项目中已经存在的 logo 素材，文案保持中性，并避免披露合作细节。
- [Risk] 客户 logo 原始比例差异较大，导致视觉重量不一致 → Mitigation：通过固定 logo 容器、`object-fit: contain`、最大宽高约束统一展示。
- [Risk] 新增区块导致页面变长，影响预约演示入口曝光 → Mitigation：区块保持短小，放在 CTA 前作为转化前的信任增强，而不是长篇案例。
